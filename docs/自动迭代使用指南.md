# 自动回测迭代系统 - 使用指南

## 概述

自动回测迭代系统是一个**持续优化和验证的闭环系统**，能够自动完成：

```
回测 → 分析 → 优化 → 验证 → 决策 → 下一轮迭代
```

## 核心特性

### 1. 全自动流程

```
迭代1: 初始回测 → 识别问题 → 优化参数 → 验证效果 → 决策是否采用
迭代2: 使用新参数 → 识别问题 → 继续优化 → 验证效果 → 决策是否采用
...
迭代N: 达到目标或无法继续改进
```

### 2. 智能决策

- 自动识别问题（胜率低、收益低、回撤大）
- 自动生成优化建议
- 自动评估改进效果
- 可选自动应用或手动确认

### 3. 数据分割

```
训练集 (60%) → 用于参数优化
验证集 (20%) → 用于验证优化效果
测试集 (20%) → 用于最终测试
```

避免过拟合，确保参数泛化能力。

### 4. 完整记录

- 每次迭代的参数配置
- 每次迭代的性能指标
- 识别的问题和改进建议
- 策略文件备份
- 可视化报告

## 快速开始

### 基础用法

```bash
python quantitative_retest/auto_backtest_iteration.py \
  --strategy liquidity_grab \
  --start-date 2024-01-01 \
  --end-date 2024-12-31 \
  --max-iterations 3
```

这将：
1. 运行3次迭代
2. 每次迭代后询问是否采用新参数
3. 生成完整的迭代报告

### 全自动模式

```bash
python quantitative_retest/auto_backtest_iteration.py \
  --strategy liquidity_grab \
  --start-date 2016-01-01 \
  --end-date 2021-12-31 \
  --max-iterations 5 \
  --auto-apply
```

添加 `--auto-apply` 后，系统会自动应用改进的参数，无需手动确认。

### 仅分析模式

```bash
python quantitative_retest/auto_backtest_iteration.py \
  --strategy liquidity_grab \
  --start-date 2024-01-01 \
  --end-date 2024-12-31 \
  --max-iterations 3 \
  --disable-optimization
```

只运行回测和分析，不进行参数优化。适合快速评估当前参数。

## 完整参数说明

```bash
python auto_backtest_iteration.py [选项]

必需参数:
  --start-date DATE        回测开始日期 (YYYY-MM-DD)
  --end-date DATE          回测结束日期 (YYYY-MM-DD)

可选参数:
  --strategy NAME          策略名称 (liquidity_grab 或 wyckoff_spring)
                          默认: liquidity_grab
  
  --max-iterations N       最大迭代次数
                          默认: 5
  
  --optimization-method M  优化方法 (grid 或 bayesian)
                          默认: bayesian
  
  --auto-apply            自动应用改进的参数（不询问）
  
  --disable-optimization  禁用参数优化（仅分析）
  
  --output-dir DIR        输出目录
                          默认: quantitative_retest/iterations
```

## 工作流程详解

### 单次迭代流程

```
┌─────────────────────────────────────────────────────────────┐
│ 迭代 N                                                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│ 步骤1: 运行回测（训练集）                                   │
│   ├─ 使用当前参数                                          │
│   ├─ 记录交易明细                                          │
│   └─ 计算性能指标                                          │
│                                                             │
│ 步骤2: 智能分析                                             │
│   ├─ 识别问题（胜率低、收益低、回撤大）                    │
│   ├─ 分析交易记录                                          │
│   └─ 生成优化建议                                          │
│                                                             │
│ 步骤3: 参数优化（如果启用）                                 │
│   ├─ 定义搜索空间                                          │
│   ├─ 运行优化算法                                          │
│   └─ 找到最优参数                                          │
│                                                             │
│ 步骤4: 验证效果（验证集）                                   │
│   ├─ 使用新参数回测                                        │
│   ├─ 对比改进幅度                                          │
│   └─ 评估泛化能力                                          │
│                                                             │
│ 步骤5: 决策                                                 │
│   ├─ 如果改进显著 → 采用新参数                             │
│   ├─ 如果改进不明显 → 保持原参数                           │
│   └─ 备份策略文件                                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 多次迭代流程

```
迭代1: 初始参数 → 回测 → 分析 → 优化 → 验证 → 采用新参数
         ↓
迭代2: 新参数 → 回测 → 分析 → 优化 → 验证 → 采用新参数
         ↓
迭代3: 新参数 → 回测 → 分析 → 优化 → 验证 → 改进不明显，保持原参数
         ↓
迭代4: 原参数 → 回测 → 分析 → 优化 → 验证 → 采用新参数
         ↓
迭代5: 新参数 → 回测 → 达到目标！
         ↓
最终测试: 在测试集上验证
```

## 最佳实践

### 1. 迭代次数设置

```
数据量充足（>200笔交易）：
  - 推荐 5-10 次迭代
  - 有足够样本支持多次优化

数据量一般（100-200笔）：
  - 推荐 3-5 次迭代
  - 避免过拟合

数据量较少（<100笔）：
  - 推荐 2-3 次迭代
  - 谨慎优化，重点分析
```

### 2. 优化方法选择

```
参数少（≤3个）：
  - 使用网格搜索
  - 保证找到最优解

参数多（3-5个）：
  - 使用贝叶斯优化
  - 高效且效果好

参数很多（>5个）：
  - 分批优化
  - 先优化影响大的参数
```

### 3. 自动应用策略

```
开发阶段：
  - 不使用 --auto-apply
  - 手动确认每次改进
  - 理解参数变化

成熟阶段：
  - 使用 --auto-apply
  - 自动化运行
  - 定期检查结果
```

### 4. 数据分割比例

```
默认 (60/20/20)：
  - 适合大多数情况
  - 平衡优化和验证

数据充足 (50/25/25)：
  - 更多验证数据
  - 更可靠的评估

数据较少 (70/15/15)：
  - 更多训练数据
  - 但验证可能不够可靠
```

### 5. 目标设置

```python
# 编辑 auto_backtest_iteration.py

# 保守目标（稳健策略）
self.target_win_rate = 0.60
self.target_return = 0.15

# 平衡目标（推荐）
self.target_win_rate = 0.65
self.target_return = 0.20

# 激进目标（高风险）
self.target_win_rate = 0.70
self.target_return = 0.30
```

## 常见问题

### Q1: 迭代后效果反而变差？

**原因：** 过拟合训练集

**解决方案：**
1. 检查验证集表现（应该与训练集接近）
2. 减少迭代次数
3. 增加训练集样本量
4. 使用更保守的参数范围

### Q2: 每次迭代改进很小？

**原因：** 参数已接近最优或搜索空间太小

**解决方案：**
1. 扩大参数搜索范围
2. 尝试优化其他参数
3. 检查是否已达到策略上限

### Q3: 测试集表现远差于训练集？

**原因：** 严重过拟合

**解决方案：**
1. 使用最保守的迭代结果
2. 重新运行，减少迭代次数
3. 检查训练集和测试集的市场环境差异

### Q4: 优化时间太长？

**解决方案：**
1. 使用贝叶斯优化（比网格搜索快10倍）
2. 减少优化参数数量
3. 缩短回测时间范围
4. 减少迭代次数

### Q5: 如何恢复之前的参数？

**方案1：** 从备份恢复

```bash
# 查看备份
ls quantitative_retest/iterations/iter*_smc_liquidity_strategy.py

# 恢复特定迭代的参数
cp quantitative_retest/iterations/iter3_smc_liquidity_strategy.py \
   smc_liquidity_strategy.py
```

**方案2：** 从报告中提取

```python

    report = json.load(f)
```

## 进阶技巧

### 1. 自定义评分函数

编辑 `auto_backtest_iteration.py`:

```python
class IterationRecord:
    def _calculate_score(self, metrics: Dict) -> float:
        # 自定义评分公式
        return (
            metrics.get('total_return', 0) * 40 +  # 更重视收益
            metrics.get('win_rate', 0) * 20 +      # 降低胜率权重
            metrics.get('avg_return', 0) * 20 -
            abs(metrics.get('max_drawdown', 0)) * 20
        )
```

## 总结

### 核心优势

1. **全自动** - 无需人工干预，自动完成优化闭环
2. **智能决策** - 自动识别问题，生成优化建议
3. **避免过拟合** - 训练集/验证集/测试集三重验证
4. **完整记录** - 每次迭代的参数、指标、文件全部保存
5. **可视化** - 直观展示优化进度和效果

### 推荐工作流

```
1. 初次运行（手动确认）
   python auto_backtest_iteration.py --max-iterations 3

2. 查看结果
   python visualize_iterations.py --show-params

3. 满意后自动运行
   python auto_backtest_iteration.py --max-iterations 5 --auto-apply

4. 定期重新优化（每季度）
   设置定时任务自动运行
```

### 注意事项

- ⚠️ 迭代次数不是越多越好（避免过拟合）
- ⚠️ 测试集表现才是真实表现
- ⚠️ 定期重新优化（市场环境会变化）
- ⚠️ 备份重要的参数配置
