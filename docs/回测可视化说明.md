# 回测可视化工具使用说明

## 功能概述

`visualize_backtest.py` 是一个强大的回测可视化工具，支持以下功能：

1. **绘制K线图**：显示股票价格走势
2. **标注交易点**：清晰标记买入点和卖出点
3. **绘制趋势线**：显示长期和短期的支撑/阻力趋势线
4. **显示止损和目标价**：水平线标注止损和目标价位
5. **模拟回测**：根据买入点自动计算卖出点
6. **批量可视化**：批量生成多个交易的图表

## 使用方式

### 方式一：手动指定买卖点

适用于已知买入和卖出信息的场景。

#### 基本用法

```bash
# 指定买入和卖出日期
python visualize_backtest.py --stock 600519 --buy-date 2024-01-15 --sell-date 2024-02-20

# 指定买入价格、止损和目标价
python visualize_backtest.py --stock 000001 --buy-date 2024-01-15 --buy-price 12.5 --stop-loss 11.8 --target 14.0

# 保存图表到文件
python visualize_backtest.py --stock 600519 --buy-date 2024-01-15 --sell-date 2024-02-20 --save output.png

# 不显示趋势线
python visualize_backtest.py --stock 600519 --buy-date 2024-01-15 --no-trendlines
```

#### 参数说明

- `--stock`: 股票代码（必需）
- `--buy-date`: 买入日期，格式 YYYY-MM-DD（必需）
- `--sell-date`: 卖出日期，格式 YYYY-MM-DD（可选）
- `--buy-price`: 买入价格（可选，默认使用开盘价）
- `--sell-price`: 卖出价格（可选，默认使用开盘价）
- `--stop-loss`: 止损价（可选）
- `--target`: 目标价（可选）
- `--exit-reason`: 卖出原因（可选）
- `--days-before`: 买入前显示的天数（默认60天）
- `--days-after`: 卖出后显示的天数（默认30天）
- `--save`: 保存路径（可选，不指定则显示图表）
- `--no-trendlines`: 不显示趋势线（可选）

### 方式二：模拟回测（自动计算卖出点）

适用于只知道买入点，需要模拟回测过程计算卖出点的场景。

#### 基本用法

```bash
# 基本模拟回测
python visualize_backtest.py --stock 600519 --buy-date 2024-01-15 --simulate

# 指定买入价格
python visualize_backtest.py --stock 000001 --buy-date 2024-01-15 --buy-price 12.5 --simulate

# 指定最大持仓天数
python visualize_backtest.py --stock 600519 --buy-date 2024-01-15 --simulate --max-holding-days 30

# 指定止损和目标价
python visualize_backtest.py --stock 000001 --buy-date 2024-01-15 --stop-loss 11.8 --target 14.0 --simulate

# 保存结果
python visualize_backtest.py --stock 600519 --buy-date 2024-01-15 --simulate --save result.png
```

#### 模拟回测参数

- `--simulate`: 启用模拟回测模式（必需）
- `--max-holding-days`: 最大持仓天数（默认60天）
- 其他参数同方式一

#### 模拟回测逻辑

模拟回测会根据以下规则自动计算卖出点：

1. **跌破支撑**：价格跌破趋势线支撑（主要卖出条件）
2. **超过最大持仓天数**：持仓时间超过设定值（避免无限持仓）

如果未指定止损和目标价，系统会根据趋势线分析自动计算：
- **止损价**：短期支撑线下方（根据趋势强度调整缓冲）
- **目标价**：买入价上方8-12%（根据趋势强度调整）

注意：止损和目标价仅用于图表显示，不作为实际卖出条件。

### 方式三：批量可视化回测结果

适用于从回测引擎获取多个交易记录，批量生成图表的场景。

#### Python代码示例

```python
from visualize_backtest import BacktestVisualizer

# 创建可视化工具
visualizer = BacktestVisualizer()

# 准备交易记录列表
trades = [
    {
        'stock_code': '600519',
        'buy_date': '2024-01-15',
        'buy_price': 1650.0,
        'sell_date': '2024-02-20',
        'sell_price': 1720.0,
        'stop_loss': 1600.0,
        'target': 1750.0,
        'exit_reason': '达到目标'
    },
    {
        'stock_code': '000001',
        'buy_date': '2024-01-10',
        'buy_price': 12.5,
        'sell_date': '2024-02-15',
        'sell_price': 11.8,
        'stop_loss': 11.8,
        'target': 14.0,
        'exit_reason': '止损'
    },
    # ... 更多交易记录
]

# 批量生成图表
visualizer.visualize_backtest_results(
    trades=trades,
    output_dir='backtest_charts',  # 输出目录
    max_charts=20  # 最多生成20张图表
)
```

## 图表说明

生成的图表包含以下元素：

### 主图（K线图）
- **红色K线**：收盘价 ≥ 开盘价（上涨）
- **绿色K线**：收盘价 < 开盘价（下跌）
- **蓝色虚线**：长期支撑线（连接低点，基于买入前数据）
- **橙色虚线**：长期阻力线（连接高点，基于买入前数据）
- **青色虚线**：短期支撑线（基于买入前数据）
- **紫色虚线**：短期阻力线（基于买入前数据）
- **点线延伸**：趋势线从买入点延伸到卖出点的预测部分
- **红色三角形**：买入点
- **绿色/深红倒三角**：卖出点（绿色=盈利，深红=亏损）
- **红色虚线**：止损价
- **绿色虚线**：目标价

**趋势线说明**：
- 趋势线基于买入前的历史数据计算（实线/虚线部分）
- 从买入点到卖出点的延伸部分用点线表示（预测延伸）
- 如果启用卖出时趋势线对比，会用较淡的实线显示卖出时的趋势线

### 副图（成交量）
- **红色柱**：当天上涨的成交量
- **绿色柱**：当天下跌的成交量

### 场景3：分析回测结果

```python
# 从回测引擎读取交易记录
from quantitative_retest.backtest_engine import BacktestEngine

engine = BacktestEngine(initial_capital=100000)
engine.run('2024-01-01', '2024-12-31', 'liquidity_grab')

# 转换为可视化格式
trades = []
for trade in engine.trades:
    trades.append({
        'stock_code': trade.stock_code,
        'buy_date': trade.buy_date,
        'buy_price': trade.buy_price,
        'sell_date': trade.sell_date,
        'sell_price': trade.sell_price,
        'exit_reason': trade.exit_reason
    })

# 批量生成图表
from visualize_backtest import BacktestVisualizer
visualizer = BacktestVisualizer()
visualizer.visualize_backtest_results(trades, output_dir='backtest_results')
```

### 场景4：对比不同止损策略

## 注意事项

1. **数据要求**：确保数据库中有足够的历史数据（买入前至少60天）
2. **日期格式**：所有日期必须使用 YYYY-MM-DD 格式
3. **交易日**：买入和卖出日期必须是交易日
4. **价格合理性**：手动指定的价格应在当天的最高价和最低价之间
5. **趋势线计算**：趋势线基于买入前的历史数据计算，不包含买入后的数据

## 常见问题

### Q1: 为什么看不到趋势线？
A: 可能是历史数据不足（少于30天）或趋势线不明显。可以增加 `--days-before` 参数。

### Q2: 模拟回测的卖出点不准确？
A: 模拟回测使用的是简化的卖出逻辑，实际回测引擎可能有更复杂的规则。可以调整 `--stop-loss`、`--target` 和 `--max-holding-days` 参数。

### Q3: 如何批量处理多个股票？
A: 使用 Python 脚本调用 `visualize_backtest_results()` 方法，或编写 shell 脚本循环调用命令行工具。

### Q4: 图表太小看不清？
A: 可以修改代码中的 `figsize=(16, 10)` 参数来调整图表大小。

## 技术细节

### 趋势线计算方法

- **长期趋势线**：使用120天数据，分成5段，每段找最低点/最高点
- **短期趋势线**：使用30天数据，分成5段，每段找最低点/最高点
- **趋势强度**：综合考虑斜率、触点数量、持续时间、价格一致性等因素

### 止损和目标价自动计算

```python
# 止损价 = 短期支撑线 * (1 - 缓冲)
# 缓冲根据趋势强度调整：
# - 强趋势（80+）：5-6%
# - 中等趋势（60-80）：4-5%
# - 弱趋势（40-60）：3-4%
# - 很弱（<40）：3%

# 目标价 = 买入价 * (1 + 涨幅)
# 涨幅根据趋势强度调整：
# - 强趋势（70+）：12%
# - 中等趋势（50-70）：10%
# - 弱趋势（<50）：8%
```
