# 参数优化使用指南

## 概述

本系统提供三种参数优化方式：

1. **智能分析器** - 从历史回测报告中学习，给出参数调整建议
2. **自动优化器** - 使用网格搜索或贝叶斯优化自动寻找最优参数
3. **手动调优** - 根据经验手动调整参数

## 一、智能分析器（推荐入门使用）

### 功能
- 分析历史交易记录，识别问题模式
- 统计不同置信度区间的实际胜率
- 给出针对性的参数调整建议

### 使用方法

```bash
# 分析历史回测报告
python quantitative_retest/smart_parameter_tuner.py backtest_trades_liquidity_grab_20260121_220321.csv
```

### 输出示例

- 无法找到全局最优解

## 二、自动优化器（推荐进阶使用）

### 功能
- 自动搜索参数空间
- 找到最优参数组合
- 支持网格搜索和贝叶斯优化

### 使用方法

#### 方法1: 网格搜索（全面但慢）

```bash
python quantitative_retest/auto_optimize_and_backtest.py \
  --enable-optimization \
  --optimization-method grid \
  --optimization-iterations 50 \
  --start-date 2024-01-01 \
  --end-date 2024-12-31 \
  --strategy liquidity_grab
```

#### 方法2: 贝叶斯优化（高效推荐）

```bash
# 需要先安装依赖
pip install scikit-optimize

# 运行优化
python quantitative_retest/auto_optimize_and_backtest.py \
  --enable-optimization \
  --optimization-method bayesian \
  --optimization-iterations 30 \
  --start-date 2024-01-01 \
  --end-date 2024-12-31 \
  --strategy liquidity_grab
```

### 工作流程

```
1. 数据分割
   ├─ 训练集（前80%）: 用于参数优化
   └─ 测试集（后20%）: 用于验证效果

2. 参数优化
   ├─ 定义搜索空间
   ├─ 生成参数组合
   ├─ 运行回测
   └─ 计算评分

3. 参数更新
   ├─ 选择最优参数
   ├─ 更新策略文件
   └─ 重新加载模块

4. 完整回测
   ├─ 使用最优参数
   ├─ 运行全量回测
   └─ 生成报告
```

### 评分公式

```python
Score = 总收益率 * 0.3 + 胜率 * 0.3 + 夏普比率 * 0.2 - 最大回撤 * 0.2
```

### 优化参数列表

当前支持优化的参数：

| 参数名 | 说明 | 默认值 | 搜索范围 |
|--------|------|--------|----------|
| MIN_CONFIDENCE | 最低置信度 | 80 | 70-90 |
| MIN_ENTRY_QUALITY | 最低入场质量 | 85 | 75-95 |
| MIN_TREND_STRENGTH | 最低趋势强度 | 80 | 70-90 |
| ATR_STOP_MULTIPLIER | ATR止损倍数 | 1.5 | 1.0-2.5 |
| ATR_TARGET_MULTIPLIER | ATR目标倍数 | 3.0 | 2.0-4.0 |
| LIQUIDITY_SWEEP_THRESHOLD | 流动性扫荡阈值 | 0.02 | 0.01-0.03 |
| ORDER_BLOCK_STRENGTH | 订单块强度 | 1.5 | 1.2-2.0 |

### 优势
- 自动化程度高
- 可以找到更优解
- 避免主观偏见

### 局限
- 计算时间较长
- 可能过拟合训练集
- 需要足够的历史数据

## 四、参数优化最佳实践

### 1. 避免过拟合

```python
# ✅ 正确做法：使用训练集优化，测试集验证
训练集: 2024-01-01 至 2024-09-30 (优化参数)
测试集: 2024-10-01 至 2024-12-31 (验证效果)

# ❌ 错误做法：在全部数据上优化
全部数据: 2024-01-01 至 2024-12-31 (优化参数)
```

### 2. 样本量要求

- 最少：30笔交易
- 推荐：100笔以上交易
- 理想：200笔以上交易

### 3. 参数稳定性测试

```bash
# 在不同时间段测试参数稳定性
python quantitative_retest/run_backtest.py --start-date 2023-01-01 --end-date 2023-12-31
python quantitative_retest/run_backtest.py --start-date 2024-01-01 --end-date 2024-12-31
python quantitative_retest/run_backtest.py --start-date 2025-01-01 --end-date 2025-03-31
```

如果参数在不同时间段表现差异很大，说明过拟合。

### 4. 多目标优化

不要只追求单一指标：

```python
# ❌ 只看收益率
最优参数: 收益率 50%，但胜率只有 30%，最大回撤 -40%

# ✅ 综合评估
最优参数: 收益率 25%，胜率 65%，最大回撤 -10%
```

### 5. 定期重新优化

市场环境会变化，建议：
- 每季度重新分析一次
- 每半年重新优化一次
- 发现策略失效时立即优化

## 五、常见问题

### Q1: 优化后效果反而变差？

**原因**: 过拟合训练集

**解决方案**:
1. 增加训练集样本量
2. 减少优化参数数量
3. 使用更保守的参数范围
4. 在测试集上验证

### Q2: 优化时间太长？

**解决方案**:
1. 使用贝叶斯优化代替网格搜索
2. 减少优化迭代次数
3. 减少优化参数数量
4. 缩短回测时间范围

### Q3: 如何选择优化参数？

**优先级**:
1. 高优先级：置信度阈值、入场质量
2. 中优先级：止损止盈倍数
3. 低优先级：技术指标周期

**原则**: 先优化影响最大的参数

### Q4: 优化结果不稳定？

**检查清单**:
- [ ] 样本量是否足够（>100笔）
- [ ] 是否使用了训练集/测试集分割
- [ ] 参数范围是否合理
- [ ] 评分公式是否合理

## 六、进阶技巧

### 1. 自定义评分函数

编辑 `parameter_optimizer.py`:

```python
def calculate_score(self, metrics: Dict) -> float:
    """自定义评分函数"""
    # 示例：更重视胜率
    return (
        metrics['total_return'] * 0.2 +  # 收益率权重降低
        metrics['win_rate'] * 0.5 +      # 胜率权重提高
        metrics['sharpe_ratio'] * 0.2 -
        abs(metrics['max_drawdown']) * 0.1
    )
```

### 2. 添加新的优化参数

编辑 `parameter_optimizer.py`:

```python
def _define_parameter_space(self):
```

## 八、总结

### 推荐流程

```
新手 → 智能分析器 → 手动调整 → 验证
进阶 → 网格搜索 → 自动优化 → 验证
高级 → 贝叶斯优化 → 多目标优化 → 持续监控
```

### 关键原则

1. **数据驱动**: 基于历史数据，不凭感觉
2. **避免过拟合**: 训练集优化，测试集验证
3. **综合评估**: 不只看收益率，要看胜率、回撤
4. **持续优化**: 定期重新评估和调整
5. **风险控制**: 优化不能以牺牲风控为代价
