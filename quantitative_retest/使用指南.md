# 股票交易策略回测系统 - 使用指南

## 快速开始

### 第一步：确保环境准备就绪

1. **检查Python版本**（需要Python 3.8+）
```bash
python --version
```

2. **安装依赖包**
```bash
pip install -r requirements.txt
```

3. **确保数据库已初始化**

回测系统需要本地数据库包含近3年的股票历史数据。如果尚未初始化，请先运行：
```bash
python main.py
```
选择"初始化数据库"选项。

### 第二步：测试模块导入

```bash
cd quantitative_retest
python test_import.py
```

如果看到"✓ 所有模块导入成功！"，说明环境配置正确。

### 第三步：运行回测

```bash
python run_backtest.py
```

按照提示选择：
1. 回测策略（SMC流动性猎取 或 威科夫Spring反转）
2. 时间范围（默认近3年，或自定义）

### 第四步：查看结果

回测完成后，系统会生成：
- **控制台报告**：显示总收益率、胜率、交易统计
- **CSV文件**：详细的交易记录
- **PNG图表**：资金曲线和收益率曲线

## 详细说明

### 回测策略

#### 1. SMC流动性猎取策略 (liquidity_grab)

**特点**：
- 捕捉机构"扫止损"后的快速反弹
- 适合短期交易
- 高胜率策略

**适用场景**：
- 市场波动较大时
- 寻找快速进出机会
- 追求高胜率

#### 2. 威科夫Spring反转策略 (wyckoff_spring)

**特点**：
- 基于40年K线统计数据
- 识别底部反转形态
- 统计学优势

**适用场景**：
- 寻找底部反转机会
- 长期持有
- 基于概率优势

### 回测参数

#### 初始资金
- 默认：1.0（归一化资金）
- 可以理解为100%的资金

#### 手续费率
- 默认：1%（买入和卖出各1%）
- 模拟真实交易成本

#### 时间范围
- 默认：近3年
- 可自定义开始和结束日期

### 交易规则

#### 买入规则
1. 每个交易日获取策略信号
2. 选择置信度最高的信号
3. 在信号日的**下一个交易日**以**开盘价**买入
4. 使用**全部可用资金**买入
5. 扣除1%买入手续费

#### 卖出规则
1. 每个交易日检查持仓
2. **止损触发**：当日最低价 ≤ 止损价
3. **止盈触发**：当日最高价 ≥ 目标价
4. **开盘直接触发**：
   - 开盘价 ≤ 止损价 → 以止损价卖出
   - 开盘价 ≥ 目标价 → 以目标价卖出
5. 扣除1%卖出手续费

#### 持仓管理
- **单一持仓**：同一时间只持有一只股票
- **全仓交易**：每次使用全部可用资金
- **严格止损止盈**：自动执行，无人工干预

### 性能指标说明

#### 总收益率
```
总收益率 = (最终资金 - 初始资金) / 初始资金
```
例如：初始1.0，最终1.5，则收益率为50%

#### 胜率
```
胜率 = 盈利交易数 / 总交易数
```
例如：10笔交易，6笔盈利，胜率为60%

#### 平均盈利/亏损
```
平均盈利 = 所有盈利交易的平均值
平均亏损 = 所有亏损交易的平均值
```

#### 最大盈利/亏损
单笔交易的最大盈利和最大亏损

### 输出文件说明

#### 交易记录CSV
文件名格式：`backtest_trades_{策略名}_{时间戳}.csv`

包含字段：
- 股票代码
- 买入日期、买入价
- 卖出日期、卖出价
- 股数
- 买入手续费、卖出手续费
- 净利润
- 收益率
- 卖出原因（stop_loss 或 target）

#### 收益率曲线图
文件名格式：`backtest_equity_curve_{策略名}_{时间戳}.png`

包含两个子图：
1. **资金曲线**：显示资金随时间的变化
2. **累计收益率曲线**：显示收益率随时间的变化

## 高级用法

### 自定义回测参数

编辑 `run_backtest.py`，修改以下参数：

```python
# 修改初始资金
engine = BacktestEngine(initial_capital=10000.0, commission_rate=0.01)

# 修改手续费率
engine = BacktestEngine(initial_capital=1.0, commission_rate=0.005)  # 0.5%
```

### 编程方式调用

```python
from backtest_engine import BacktestEngine
from performance_analyzer import PerformanceAnalyzer, ReportGenerator

# 初始化引擎
engine = BacktestEngine(initial_capital=1.0, commission_rate=0.01)

# 运行回测
engine.run('2023-01-01', '2026-01-20', 'liquidity_grab')

# 获取结果
results = engine.get_results()

# 分析性能
analyzer = PerformanceAnalyzer()
metrics = analyzer.calculate_metrics(
    results['trades'],
    results['initial_capital'],
    results['final_capital']
)

# 生成报告
report_generator = ReportGenerator()
report_generator.print_summary(metrics, '2023-01-01', '2026-01-20', 'SMC流动性猎取')
```

### 批量回测

创建脚本批量测试不同参数：

## 常见问题

### Q1: 为什么没有交易记录？

**可能原因**：
1. 数据库数据不足（少于30个交易日）
2. 策略没有找到符合条件的信号
3. 时间范围设置不当

**解决方法**：
1. 检查数据库：`python check_database.py`
2. 扩大时间范围
3. 降低策略的置信度阈值（修改 `integrated_screener.py` 中的 `MINCONFIDENCE`）

### Q2: 回测速度很慢？

**优化建议**：
1. 缩小时间范围
2. 减少候选股票数量
3. 使用更快的硬盘（SSD）

### Q3: 收益率为负数？

这是正常现象，说明策略在该时间段表现不佳。可以：
1. 尝试不同的时间段
2. 尝试不同的策略
3. 分析亏损交易，优化策略参数

### Q4: 如何验证回测的正确性？

系统实现了严格的验证机制：
1. **时间序列严格性**：不使用未来数据
2. **单一持仓约束**：同时只持有一只股票
3. **手续费一致性**：所有交易都扣除手续费
4. **资金守恒**：资金变化可追溯

可以手工验证几笔交易，确保逻辑正确。

## 注意事项

### 1. 数据质量
- 确保数据库包含完整的历史数据
- 定期更新数据库
- 检查数据的准确性

### 2. 回测局限性
- **历史表现不代表未来**
- 回测结果可能存在过拟合
- 真实交易有滑点、流动性等问题

### 3. 风险提示
- 本系统仅供学习和研究
- 不构成投资建议
- 实盘交易需谨慎

### 4. 性能考虑
- 大量股票回测需要较长时间
- 建议先用小范围测试
- 可以使用进度显示监控进度
