# 股票交易策略回测系统

## 概述

本系统用于验证已实现的选股策略（SMC流动性猎取和威科夫Spring模型）的历史表现，通过模拟真实交易来评估策略的收益率和胜率。

## 功能特性

- ✅ 从本地数据库加载近3年历史数据
- ✅ 支持两种策略：SMC流动性猎取、威科夫Spring反转
- ✅ 严格防止未来数据泄露
- ✅ 真实交易模拟（包含1%手续费）
- ✅ 自动止损/止盈执行
- ✅ 详细的性能指标统计
- ✅ 收益率曲线可视化
- ✅ 交易记录CSV导出

## 使用方法

### 1. 确保数据库已初始化

回测系统需要本地数据库包含近3年的股票历史数据。如果尚未初始化，请先运行：

```bash
python main.py
```

选择"初始化数据库"选项。

### 2. 运行回测

```bash
cd quantitative_retest
python run_backtest.py
```

### 3. 选择策略

系统会提示选择回测策略：
- 1: SMC流动性猎取策略（快速反弹，高胜率）
- 2: 威科夫Spring反转策略（底部反转，统计优势）

### 4. 设置时间范围

可以使用默认的近3年时间范围，或自定义开始和结束日期。

### 5. 查看结果

回测完成后，系统会生成：
- 控制台摘要报告（总收益率、胜率、交易统计）
- CSV交易记录文件
- PNG收益率曲线图

## 回测逻辑

### 买入规则

1. 每个交易日调用策略接口获取交易信号
2. 只使用当前日期之前的历史数据（防止未来数据泄露）
3. 选择置信度最高的信号
4. 在信号日的下一个交易日以开盘价买入
5. 使用全部可用资金买入
6. 扣除1%买入手续费

### 卖出规则

1. 每个交易日检查持仓是否触发止损或止盈
2. 如果开盘价低于止损价，以止损价卖出
3. 如果开盘价高于目标价，以目标价卖出
4. 如果开盘价在区间内，检查当日最低价是否触及止损，或最高价是否触及目标
5. **新增**：如果持仓股票连续3天收盘价未上涨，以收盘价卖出
6. 扣除1%卖出手续费

### 性能优化

- **智能信号检测**：持仓且当天不卖出时，跳过买入信号检测，提升回测速度

### 资金管理

- 初始资金：1.0
- 单一持仓：同一时间只持有一只股票
- 全仓交易：每次使用全部可用资金
- 手续费：买入和卖出各扣除1%

## 性能指标

系统计算以下性能指标：

- **总收益率**: (最终资金 - 初始资金) / 初始资金
- **胜率**: 盈利交易数 / 总交易数
- **总交易次数**: 完成的交易笔数
- **盈利/亏损交易数**: 分别统计
- **平均盈利/亏损**: 单笔交易的平均盈亏
- **最大盈利/亏损**: 单笔交易的最大盈亏

## 输出文件

### 交易记录CSV

包含以下字段：
- 股票代码
- 买入日期、买入价
- 卖出日期、卖出价
- 股数
- 买入/卖出手续费
- 净利润
- 收益率
- 卖出原因（止损/止盈）

### 收益率曲线图

包含两个子图：
1. 资金曲线：显示资金随时间的变化
2. 累计收益率曲线：显示收益率随时间的变化

## 注意事项

1. **数据完整性**: 股票数据少于30个交易日会被跳过
2. **时间序列**: 严格按日期升序处理，防止未来数据泄露
3. **单一持仓**: 有持仓时会忽略新的买入信号
4. **真实成本**: 所有交易都包含1%手续费

## 技术实现

### 核心类

#### BacktestEngine
- 主控制器，协调所有组件
- 管理回测流程和状态

#### DataLoader
- 从SQLite数据库加载历史数据
- 验证数据完整性
- 提供交易日列表

#### StrategyInterface
- 与IntegratedScreener交互
- 获取交易信号
- 选择最高置信度信号

#### PositionManager
- 管理持仓状态
- 执行买入/卖出操作
- 检查止损/止盈触发

#### PerformanceAnalyzer
- 计算性能指标
- 生成收益率曲线

#### ReportGenerator
- 输出控制台报告
- 保存CSV和图片文件
