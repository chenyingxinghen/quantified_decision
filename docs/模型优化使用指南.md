# 模型优化使用指南

## 概述

本指南介绍如何使用模型优化工具进行：
1. **特征选择** - 从40+个因子中选择最有效的因子
2. **特征工程超参数优化** - 优化因子计算周期等参数
3. **集成学习优化** - 优化模型权重，提升集成效果

---

## 快速开始

### 1. 运行完整优化流程

```bash
python scripts/run_model_optimization.py
```

这将执行：
- 加载训练数据
- 训练基础模型（XGBoost、LightGBM、Random Forest）
- 特征选择（从40+个因子中选择最优的30个）
- 集成权重优化
- 保存优化结果

### 2. 查看优化结果

优化完成后，会生成以下文件：

```
models/
├── optimization_results.json      # 优化结果（JSON格式）
├── optimization_report.txt        # 优化报告（文本格式）
├── selected_features.txt          # 选择的特征列表
└── optimized_ensemble_model.pkl   # 优化后的集成模型
```

## 详细说明

### 一、特征选择

#### 1.1 支持的方法

| 方法 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **importance** | 基于模型特征重要性 | 快速、直观 | 依赖单个模型 |
| **correlation** | 基于与目标的相关性 | 简单、稳定 | 只考虑线性关系 |
| **mutual_info** | 基于互信息 | 捕捉非线性关系 | 计算较慢 |
| **rfe** | 递归特征消除 | 考虑特征交互 | 非常耗时 |
| **hybrid** | 混合方法（推荐） | 综合多种方法 | 计算量较大 |

#### 1.2 使用示例

```python
from core.factors.model_optimizer import FeatureSelector

# 创建特征选择器
selector = FeatureSelector(method='hybrid')

# 执行特征选择
selected_features = selector.select(
    model=trained_model.model,
    X=X_train,
    y=y_train,
    feature_names=factor_names,
    n_features=30,              # 选择30个特征
    remove_correlated=True,     # 移除高度相关的特征
    corr_threshold=0.95         # 相关性阈值
)

print(f"选择的特征: {selected_features}")
```

### 二、特征工程超参数优化

#### 2.1 可优化的参数

```python
param_space = {
    'rsi_period': [6, 9, 12, 14],      # RSI周期
    'macd_fast': [8, 12, 16],          # MACD快线周期
    'macd_slow': [20, 26, 32],         # MACD慢线周期
    'bb_period': [15, 20, 25],         # 布林带周期
    'atr_period': [10, 14, 20],        # ATR周期
}
```

#### 2.2 使用示例

```python
from core.factors.model_optimizer import FeatureEngineeringOptimizer

optimizer = FeatureEngineeringOptimizer()

# 优化因子计算周期
best_params = optimizer.optimize_factor_periods(
    data=stock_data,
    factor_calculator=factor_calculator,
    y=labels,
    model_class=XGBClassifier
)

print(f"最优参数: {best_params}")
```

#### 2.3 注意事项

⚠️ **特征工程超参数优化非常耗时**，因为需要：
1. 重新计算所有因子
2. 重新训练模型
3. 评估每个参数组合

建议：
- 先使用默认参数训练模型
- 如果性能不理想，再考虑优化
- 使用较小的参数搜索空间
- 使用缓存机制避免重复计算

### 三、集成学习优化

#### 3.1 支持的方法

##### 方法1: 网格搜索

```python
from core.factors.model_optimizer import EnsembleOptimizer

optimizer = EnsembleOptimizer()

# 网格搜索优化权重
best_weights = optimizer.optimize_weights_grid_search(
    models=[model1, model2, model3],
    X_val=X_validation,
    y_val=y_validation
)

print(f"最优权重: {best_weights}")
# 输出: [0.4, 0.35, 0.25]
```

**优点**：
- 简单直观
- 保证找到网格内的最优解

**缺点**：
- 模型数量多时计算量大
- 搜索精度受网格密度限制

##### 方法2: Scipy优化

```python
# 使用scipy优化权重（更精确）
best_weights = optimizer.optimize_weights_scipy(
    models=[model1, model2, model3],
    X_val=X_validation,
    y_val=y_validation
)

print(f"最优权重: {best_weights}")
# 输出: [0.423, 0.341, 0.236]
```

**优点**：
- 更精确的权重
- 支持更多模型

**缺点**：
- 需要安装scipy
- 可能陷入局部最优

##### 方法3: Stacking

```python
# Stacking集成
stacking_result = optimizer.optimize_stacking(
    models=[model1, model2, model3],
    X_train=X_train,
    y_train=y_train,
    X_val=X_val,
    y_val=y_val
)

print(f"Stacking AUC: {stacking_result['auc']:.4f}")
print(f"元模型权重: {stacking_result['weights']}")
```

**优点**：
- 可以学习非线性组合
- 通常性能最好

**缺点**：
- 更复杂
- 需要更多数据

## 完整示例

### 示例1: 基础优化流程

```python
from core.factors.train_ml_model import MLModelTrainer
from core.factors.model_optimizer import ModelOptimizer

# 1. 训练基础模型
trainer = MLModelTrainer()
stocks_data = trainer.load_training_data(stock_codes, start_date, end_date)
X, y, factor_names = trainer.prepare_dataset(stocks_data)
results = trainer.train_models(X, y, factor_names)

# 2. 运行优化
optimizer = ModelOptimizer()
optimization_results = optimizer.run_full_optimization(
    models=trainer.models,
    X=X,
    y=y,
    feature_names=factor_names
)

# 3. 使用优化结果
selected_features = optimization_results['selected_features']
ensemble_weights = optimization_results['ensemble_weights_grid']

print(f"选择了 {len(selected_features)} 个特征")
print(f"集成权重: {ensemble_weights}")
```

### 示例2: 单独使用特征选择

```python
from core.factors.model_optimizer import FeatureSelector

# 训练一个模型
model = MLFactorModel(model_type='xgboost')
model.train(X, y, feature_names=factor_names)

# 特征选择
selector = FeatureSelector(method='hybrid')
selected = selector.select(
    model.model, X, y, factor_names,
    n_features=30
)

# 使用选择的特征重新训练
X_selected = X[:, [factor_names.index(f) for f in selected]]
model_optimized = MLFactorModel(model_type='xgboost')
model_optimized.train(X_selected, y, feature_names=selected)
```

### 示例3: 单独优化集成权重

```python
from core.factors.model_optimizer import EnsembleOptimizer

    models, X_val, y_val
```

## 常见问题

### Q1: 特征选择后性能反而下降？

**可能原因**：
- 选择的特征数太少
- 移除了某些关键特征
- 模型过拟合到训练集

**解决方案**：
```python
# 尝试不同的特征数量
for n in [20, 30, 40, 50]:
    selected = selector.select(model, X, y, factor_names, n_features=n)
    # 评估性能
```

### Q2: 集成权重优化没有提升？

**可能原因**：
- 模型之间差异太小
- 验证集太小
- 模型已经过拟合

**解决方案**：
- 使用更多样化的模型
- 增加验证集大小
- 使用时间序列交叉验证

### Q3: 优化过程太慢？

**解决方案**：
```python
# 1. 减少股票数量
stock_codes = stock_codes[:50]  # 只用50只股票

# 2. 减少特征数量
selector.select(model, X, y, factor_names, n_features=20)

# 3. 使用网格搜索而不是scipy优化
optimizer.optimize_weights_grid_search(models, X_val, y_val)

# 4. 跳过特征工程超参数优化
```

## 最佳实践

### 1. 优化顺序

```
第一步: 特征选择
  ↓
第二步: 使用选择的特征重新训练模型
  ↓
第三步: 集成权重优化
  ↓
第四步: 验证和回测
  ↓
第五步: (可选) 特征工程超参数优化
```

### 2. 定期重新优化

```python
# 建议每季度重新优化一次
# 因为市场环境会变化，因子有效性会改变

# 设置定期任务
import schedule

def optimize_models():
    optimizer = ModelOptimizer()
    results = optimizer.run_full_optimization(...)
    optimizer.save_results()

# 每季度运行一次
schedule.every(3).months.do(optimize_models)
```
