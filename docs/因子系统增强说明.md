# 因子系统增强说明

## 概述

本次更新对量化因子系统进行了全面增强，完整利用了 stock_info_extended 表的数据，并添加了先进的特征工程功能。

## 更新内容

### 1. 扩展基本面因子 (fundamental_factors.py)

#### 新增因子类别

**现金流因子 (4个)**
- `ocf_to_market_cap`: 经营现金流/市值
- `cash_to_market_cap`: 现金充裕度
- `ocf_to_debt`: 现金流与债务比
- `fcf_yield`: 自由现金流收益率

**股本结构因子 (4个)**
- `held_percent_insiders`: 内部人持股比例
- `held_percent_institutions`: 机构持股比例
- `float_ratio`: 流通比例
- `total_held_percent`: 机构+内部人持股总比例

**流动性因子 (5个)**
- `average_volume`: 平均成交量
- `average_volume_10days`: 10日平均成交量
- `turnover_ratio`: 换手率
- `avg_amount`: 平均成交金额
- `liquidity_ratio`: 流动性比率

**规模因子 (3个)**
- `log_market_cap`: 市值（对数）
- `log_book_value`: 账面价值（对数）
- `log_enterprise_value`: 企业价值（对数）

**交叉因子 (8个)**
- `roe_growth`: ROE × 营收增长
- `pe_earnings_growth`: PE × 盈利增长
- `roe_to_pb`: ROE / PB
- `roe_to_leverage`: ROE / 资产负债率
- `fcf_to_net_income`: 自由现金流 / 净利润
- `mcap_to_revenue_growth`: 市值 / 营收增长
- `dividend_quality`: 股息率 × (1 - 派息比率)
- `institution_quality`: 机构持股 × ROE

### 2. 集成基本面因子到模型训练 (train_ml_model.py)

#### 主要改动

1. **添加基本面因子计算器**
   ```python
   self.fundamental_calculator = FundamentalFactors(db_path)
   ```

2. **在因子计算中集成基本面数据**
   - 技术指标因子（时间序列）
   - K线形态因子（时间序列）
   - 基本面因子（横截面，扩展到所有时间点）

3. **更新因子汇总统计**
   - 分别统计技术指标、K线形态、基本面和工程化因子
   - 保存详细的因子名称列表

#### 数据处理逻辑

```python
# 基本面因子是横截面数据（每只股票一个值）
fundamental_dict = self.fundamental_calculator.calculate_all_fundamental_factors(code)

# 扩展到所有时间点（与技术指标对齐）
fundamental_factors = pd.DataFrame([fundamental_dict] * len(data))

# 合并所有因子
all_factors = pd.concat([tech_factors, candlestick_factors, fundamental_factors], axis=1)
```

### 3. 特征工程模块 (feature_engineering.py)

#### 核心功能

**基础变换**
- `create_ratio_features()`: 比率特征（A/B）
- `create_product_features()`: 乘积特征（A×B）
- `create_difference_features()`: 差值特征（A-B）
- `create_log_features()`: 对数变换
- `create_sqrt_features()`: 平方根变换

**高级变换**
- `create_polynomial_features()`: 多项式特征
- `create_rank_features()`: 排名特征
- `create_quantile_features()`: 分位数特征
- `create_interaction_features()`: 交互特征（技术×基本面）

**时间序列特征**
- `create_momentum_features()`: 动量特征（变化率）
- `create_volatility_features()`: 波动率特征（滚动标准差）

#### 使用示例

```python
from core.factors.feature_engineering import FeatureEngineer

engineer = FeatureEngineer()

# 应用所有变换
df_enhanced = engineer.apply_all_transformations(df, config={
    'ratio': True,
    'product': True,
    'log': True,
    'sqrt': True,
    'rank': True,
    'interaction': True,
})

# 获取生成的特征
new_features = engineer.get_generated_features()
```

## 注意事项

### 1. 数据质量

- 基本面数据可能存在缺失值（特别是PEG、5年股息等）
- 系统会自动填充0，但建议检查数据完整性
- 使用 `scripts/update_stock_info.py` 定期更新基本面数据

### 2. 计算性能

- 特征工程会显著增加特征数量（67 → 195）
- 训练时间可能增加 30-50%
- 建议使用并行计算（n_jobs=-1）
- 因子缓存机制可以加速重复训练

### 3. 过拟合风险

- 特征数量增加可能导致过拟合
- 建议使用特征选择（model_optimizer.py）
- 使用时间序列分割验证
- 监控训练集和验证集的性能差异

### 4. 特征工程配置

可以根据需要调整特征工程配置：

```python
config = {
    'ratio': True,      # 比率特征
    'product': True,    # 乘积特征
    'difference': False, # 差值特征（可选）
    'log': True,        # 对数特征
    'sqrt': True,       # 平方根特征
    'rank': True,       # 排名特征
    'interaction': True, # 交互特征
}

df_enhanced = engineer.apply_all_transformations(df, config=config)
```
